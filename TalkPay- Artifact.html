03 — System Architecture
Five-Layer System Design
Layer 1 — Voice Input
Web Speech API (browser-native) · Groq Whisper large-v3 (cloud STT) · Text input fallback · IVR planned Q3 2026
→ Output: text transcript of user command
Layer 2 — AI / NLP Processing
Groq LLM (llama-3) via pydantic-ai · Intent classification (payment/query/help) · Slot extraction (intent, amount, recipient) · Orchestrator state machine (domain/orchestrator.py)
→ Output: structured TransferRequest object
Layer 3 — Payment Orchestration
Payment state machine (idle → pending_confirm → executing → idle) · Beneficiary verification loop · Idempotency guard (MD5 → UUID, PostgreSQL UNIQUE constraint) · Confirmation dialogue management
→ Output: authorised, idempotent payment instruction
Layer 4 — Banking / API Layer
FastAPI gateway service (:8100) · Dummy payment rails (providers/dummy.py) · SQLAlchemy atomic debit+credit transactions · Supabase-hosted PostgreSQL · Balance engine · Refund/reversal endpoints
→ Output: completed transaction record
Layer 5 — Security & Authentication
Email/password login · bcrypt PIN (4-8 digits, never stored plain) · OTP challenge (single-use, replay-blocked) · Session JWT (Bearer token) · Rule-based fraud scoring (velocity + amount) · Append-only audit log · Voice biometrics — Q2 2026
→ Output: authenticated, risk-scored, logged transaction

Category Auto-Tagging
LLM tags each transaction (food, rent, salary, investment) from voice context — powers AI Advice tab.
